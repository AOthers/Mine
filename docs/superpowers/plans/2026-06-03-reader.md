# Reader Tool Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a local reader tool for novels and comics with bookshelf import, persisted progress, and basic TXT/EPUB/PDF/comic reading.

**Architecture:** Add focused reader data models, local store, import/classification services, and dedicated Compose screens. `MainActivity` only owns SAF launchers and navigation, while `ReaderViewModel` coordinates reader state and services.

**Tech Stack:** Android Kotlin, Jetpack Compose, SAF `DocumentFile`, SharedPreferences/Gson-style JSON persistence, Android `PdfRenderer`, Java `ZipFile`, existing Material3 UI patterns.
